## FAQ Added from AI acting like a Student Questions

### 1. How do we balance coding skills with using AI?
Early assignments are designed so you do not need AI. Build basic skills first. Use AI as a tutor: ask questions, request explanations, and verify your understanding.

### 2. How detailed should the workflow chart be for Assignment 1?
Show your input data files (usually 2+), expected cleaning steps, and expected exploration steps. High-level but clear.

### 3. Do I need all my datasets identified for Assignment 1?
Not necessarily. Most projects require at least two datasets. A common second dataset is census demographic or economic data.

### 4. How often should I commit changes to GitHub?
Commit often enough that commit messages clearly describe progress. Experimentation can be described in GitHub Issues rather than committed code.

### 5. When should I use GitHub Issues vs. my work log?
This is evolving. Use Issues to document experiments, errors, screenshots, and discussions. Use the work log to record your day-by-day progress. AI Agent in VS Code can be used to update issues with summaries of chat sessions.  

### 6. Should course repositories be public or private?
Instructor preference is public for transparency and replication, but GitHub Education allows students to make repos private.

### 7. How should I handle data that cannot be shared publicly?
Store raw data elsewhere. Include code that downloads or accesses the data. Keep shared files small (<100 MB) and follow data rights requirements.

## GitHub File Size Rules
- Max file size: 100 MB (hard limit)
- Warning at 50 MB
- Repo size recommended < 1 GB

## Git LFS (Large File Storage)
- Allows files up to 2â€“5 GB depending on plan

## Best Practices
- Store raw data outside the repo.
- Use scripts to download or load data.
- Use GitHub for code, documentation, workflow.
- Use external repositories (DesignSafe, Zenodo, etc.) for large data.
